# Viral Features Documentation

## Overview

Three features designed to make roast-my-codebase reports highly shareable on social media, GitHub, and team channels.

## Feature 1: ASCII Art Grade (`--ascii`)

**Purpose**: Make terminal output screenshot-worthy with big, colorful letter grades.

### Usage
```bash
roast-my-codebase --ascii
```

### Output
Displays a large 7-line ASCII art letter (A/B/C/D/F) at the top of the terminal report:

```
  ███████╗
  ██╔════╝
  █████╗
  ██╔══╝
  ██║
  ╚═╝

╔═════════════════════════════╗
║    Roast My Codebase    🔥  ║
╚═════════════════════════════╝

  Project: my-project
  Health Score: 48/100  F  Chaotic
```

### Color Mapping
- **A (90+)**: Green
- **B (80+)**: Bright Green
- **C (70+)**: Yellow
- **D (60+)**: Orange
- **F (<60)**: Red

### Use Cases
- Screenshots for Twitter/LinkedIn
- Team presentations
- Demo videos
- Social media posts

## Feature 2: SVG Badge Generation (`--badge`)

**Purpose**: Generate README badges showing codebase health scores.

### Usage
```bash
roast-my-codebase --badge
```

### Output
Creates `.roast-badge.svg` in the project root:

![Example Badge](https://img.shields.io/badge/Health-82%2F100-97ca00?style=flat)

### Embedding in README
```markdown
![Codebase Health](.roast-badge.svg)
```

Or with a link:
```markdown
[![Codebase Health](.roast-badge.svg)](https://github.com/your-org/your-repo)
```

### Badge Specifications
- **Dimensions**: 150x20 pixels
- **Format**: `Health | 82/100`
- **Font**: Verdana 11px
- **Colors**:
  - Green (#44cc11): Score >= 90
  - Yellow-green (#97ca00): Score >= 80
  - Yellow (#dfb317): Score >= 70
  - Orange (#fe7d37): Score >= 60
  - Red (#e05d44): Score < 60

### Integration Examples

**In GitHub Actions**:
```yaml
- name: Generate Health Badge
  run: npx roast-my-codebase --badge

- name: Commit Badge
  run: |
    git config user.name github-actions
    git config user.email github-actions@github.com
    git add .roast-badge.svg
    git commit -m "Update health badge [skip ci]"
    git push
```

**In Pre-commit Hook**:
```bash
#!/bin/bash
npx roast-my-codebase --badge
git add .roast-badge.svg
```

## Feature 3: Markdown Output (`--markdown` / `--markdown-file`)

**Purpose**: Generate pretty markdown for PRs, Notion, and documentation.

### Usage
```bash
# Output to stdout
roast-my-codebase --markdown

# Save to file
roast-my-codebase --markdown-file
```

### Output File
Creates `.roast-report.md` in the project root.

### Markdown Structure

### Key Features
- **Collapsible sections** (`<details>` tags) keep reports scannable
- **Emoji indicators** (🔴/⚠️/ℹ️) for quick severity assessment
- **Markdown tables** for professional stats display
- **Blockquotes** for roasts (visually distinct)
- **ASCII health bar** (visual progress indicator)
- **Info sections open by default** (critical/warnings collapsed)

## Troubleshooting

### Badge not showing in GitHub
- Ensure `.roast-badge.svg` is committed
- Check file path in markdown
- Verify SVG is valid (open in browser)

### Markdown not rendering correctly
- Some platforms require blank lines around details tags
- Test in GitHub PR first
- Escape special characters if needed

### ASCII art not colorful
- Ensure terminal supports ANSI colors
- Some CI environments disable colors
- Use `FORCE_COLOR=1` environment variable
